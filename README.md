# Artifact for the project: Universal Microservices for Monitorization

This artifact shows an implementation of monitors generated by [rmtld3synth](https://github.com/anmaped/rmtld3synth) in C++11, targeting both a **native binary** and a **WebAssembly (WASM)** module. It relies on [rtmlib](https://github.com/anmaped/rtmlib) as a git submodule for the underlying lock-free ring-buffer and monitor abstraction.

The monitors are derived from RMTLD3 (Runtime Metric Temporal Logic with Durations) formulas and evaluate timestamped event traces using a three-valued logic (T_TRUE / T_FALSE / T_UNKNOWN).

## Prerequisites

### Native build

- `g++` ≥ 7 with C++17 support
- `libpthread` and `libatomic`

### WASM build

- [WASI-SDK](https://github.com/WebAssembly/wasi-sdk) ≥ 20 (expected at `/opt/wasi-sdk`)
- [WAMR](https://github.com/bytecodealliance/wasm-micro-runtime) for running the resulting `.wasm` (`iwasm` command)

### Cloning with submodules

```bash
git clone --recurse-submodules https://github.com/<your-user>/<your-repo>.git
```

If you already cloned without `--recurse-submodules`:

```bash
git submodule update --init --recursive
```

## Building — Native

```bash
cd examples/native
g++ -std=c++17 -I../../rtmlib/src -I./generated main.cpp -o monitor -lpthread -latomic
./monitor
```

Expected output:

```
Verdict: TRUE  (formula satisfied)
```

The formula evaluated is `duration of a in 0..15 >= 5`. Events `a` at t=0 and t=9 produce a duration of 9 time units, which satisfies the ≥ 5 bound.

### Adapting to a different formula

Edit the three lines in the `/* Configuration */` block of `main.cpp`:

```cpp
#include "generated/Rtm_monitor_<id>.h"
#include "generated/Rtm_instrument_<id>.h"
#define WRITER_TYPE      Writer_rtm__<id>
#define INSTRUMENT_CLASS RTM_INSTRUMENT_<ID>
```

See [`examples/native/generate_monitors.md`](examples/native/generate_monitors.md) for instructions on how to generate new headers from a formula using the rmtld3synth Docker image.

## Building — WebAssembly (WASI)

Requires [WASI-SDK](https://github.com/WebAssembly/wasi-sdk) installed at `/opt/wasi-sdk`.

```bash
cd examples/wasm
/opt/wasi-sdk/bin/clang++ \
  --target=wasm32-wasi \
  --sysroot=/opt/wasi-sdk/share/wasi-sysroot \
  -Wall -Wextra -std=gnu++11 -DRTMLIB_ENABLE_MAP_SORT \
  -D__i386__ -D__x86__ \
  -Duseconds_t=uint32_t \
  -fno-exceptions \
  -include wasi_compat.h \
  -I../../rtmlib/src \
  main.cpp wasi_stubs.cpp -o wasm/cpptest.wasm
```

Run with WAMR:

```bash
iwasm wasm/cpptest.wasm
```

For a full explanation of each compiler flag and the WASI compatibility workarounds, see [`examples/wasm/WASM_BUILD.md`](examples/wasm/WASM_BUILD.md).

The formula evaluated is `a U[10s] b` ("proposition `a` must hold until `b` occurs, within 10 seconds"). The `main.cpp` exercises four test cases covering T_TRUE, T_FALSE, and boundary conditions.

## Generating New Monitor Headers

Monitor headers (`Rtm_compute_*.h`, `Rtm_instrument_*.h`, `Rtm_monitor_*.h`) are produced by [rmtld3synth](https://github.com/anmaped/rmtld3synth). To generate them for a new formula:

```bash
# 1. Pull the synthesis tool
docker pull anmaped/rmtld3synth

# 2. Run the synthesizer (edit the formula as needed)
docker run -it anmaped/rmtld3synth \
  rmtld3synth --synth-cpp11 \
              --input-dsl "duration of a in 0 .. 15 >= 5" \
              --out-src out

# 3. Get the container ID
docker ps -l

# 4. Copy the generated files to your target directory
docker cp <container_id>:/root/out examples/native/generated
```

See [`examples/native/generate_monitors.md`](examples/native/generate_monitors.md) for the full step-by-step procedure.

## Three-Valued Logic

All monitors return a `three_valued_type` verdict:

| Value | Meaning |
|-------|---------|
| `T_TRUE` | Formula is satisfied by the observed trace |
| `T_FALSE` | Formula is violated |
| `T_UNKNOWN` | Insufficient trace data to reach a verdict |

## Dependencies

| Dependency | Role |
|------------|------|
| [rtmlib](https://github.com/anmaped/rtmlib) | Lock-free ring-buffer, monitor thread abstraction, RMTLD3 evaluator |
| [rmtld3synth](https://github.com/anmaped/rmtld3synth) | Formula synthesizer — produces the `Rtm_*.h` headers (Docker: `anmaped/rmtld3synth`) |
| [WASI-SDK](https://github.com/WebAssembly/wasi-sdk) | C++ → wasm32-wasi cross-compiler |
| [WAMR](https://github.com/bytecodealliance/wasm-micro-runtime) | WebAssembly Micro Runtime — runs the `.wasm` binary |
